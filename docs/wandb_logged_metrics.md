# W&B Logged Metrics

This inventory enumerates concrete metric keys passed to `ExperimentTracker.log(...)` and then `wandb.log(...)`.

Dynamic-construction rule notes:
- Training debug fallback rule: if `outputs["debug"]` contains a scalar key not in `DEBUG_METRIC_MAP`, it maps to `train/model/<raw_debug_key>` via `collect_scalar_metrics()` + `map_train_diagnostic_key()`.
- Validation debug fallback rule: if evaluation debug emits a scalar not in `_VAL_DEBUG_NAMESPACE_MAP`, it maps to `val/model/<raw_key>` via `build_validation_debug_metrics()`.
- Known concrete keys generated by current code paths are fully listed below; fallback rules are included for forward compatibility with newly added debug scalars.

## Sources
- `utils/metric_logging.py` (`build_train_metrics_from_scalars`, `build_validation_metrics`)
- `processor/processor.py` (`_collect_loss_grad_norm_metrics`)
- `utils/metrics.py` (`Evaluator.eval`, `_compute_eval_debug_metrics`)
